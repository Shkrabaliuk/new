# ADR-0001: Формат зберігання документів

Дата: 2026-02-13
Статус: Proposed

## Context

Проєкт є text-first CMS, де **файлова система** — джерело істини. Нам потрібен формат, який:
- переносимий (копіювання папки = перенос контенту),
- читабельний людиною,
- добре працює з Git,
- дозволяє додавати метадані та ассети.

## Decision

Використовуємо:

1) **Контент**: `content/<slug>.md`  
2) **Метадані**: YAML Front Matter всередині `.md` (початково)  
3) **Ассети**: `assets/<slug>/...`

### Приклад `content/my-post.md`

```md
---
title: "Назва"
date: 2026-02-13
tags: [київ, архітектура]
lang: uk
draft: false
---

Тут текст...
```

## Consequences

### Плюси
- Один файл = один документ (просте версіонування).
- Сумісність з існуючими статичними генераторами.
- YAML дозволяє гнучко розширювати поля.

### Мінуси / Ризики
- YAML front matter може бути “крихким” при некоректному форматуванні.
- При зростанні метаданих може бути зручніше винести їх у `.json`.

## Alternatives considered
1) Окремий `content/<slug>.md` + `content/<slug>.json`  
2) SQLite як primary storage (відхилено через вимогу переносимості та text-first)
