# Contributing

## 1. Branches
- Кожна зміна в окремій гілці.
- Назва гілки: `feature/*`, `fix/*`, `docs/*`.

## 2. Commits
- Формат: короткий імперативний опис.
- Один логічний commit на одну задачу.

## 3. Pull requests
- Опис: мотивація, що змінено, як перевірено.
- Для змін логіки: додати/оновити тести.
- Для змін архітектури: додати або оновити ADR.

## 4. Documentation
- Будь-яка нова підсистема має мати секцію в `docs/architecture.md`.
- Зміни до вимог мають оновлювати `docs/requirements.md`.
