# Contributing

## 1. Принципи
- Маленькі PR, зрозумілий scope.
- Детерміновані трансформації тексту (golden tests).
- Документація перед кодом для ключових рішень (ADR).

## 2. Гілки
- `main` — стабільний стан.
- `feat/<topic>` — фічі.
- `fix/<topic>` — виправлення.

## 3. Коміти (conventional-ish)
Формат:
- `feat: ...`
- `fix: ...`
- `docs: ...`
- `test: ...`
- `refactor: ...`
- `chore: ...`

## 4. Pull Requests
PR має містити:
- опис проблеми/рішення,
- скрін/демо (для UI),
- тести або пояснення чому їх не додано,
- оновлення документації (якщо змінено правила pipeline/типографіки).

## 5. Код-стайл
- Лінтер/форматер — за конфігом репозиторію.
- Жодних “магічних” правил типографіки без тестів і опису.

## 6. Тести
- Unit tests: типографіка, парсер, інференс структури.
- Golden tests: вхідний `.md` → очікуваний HTML.
